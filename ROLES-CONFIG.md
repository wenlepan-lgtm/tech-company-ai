# 角色配置 - 工具与模型关联

> 更新时间: 2026-03-14

---

## 👨‍💼 小科（总控）

- **工具**: 直接执行（无需外部工具）
- **模型**: GLM-5（当前会话）
- **职责**:
  - 项目管理
  - 意图解析
  - 轻任务执行
  - 小运的工作（文档整理、日报周报）

---

## 🏗️ 小架（架构师）

- **工具**: 外部 - ChatGPT 网页版
- **模型**: GPT-4
- **流程**: 用户自己在 ChatGPT 对话 → 把 md/文档发到群里
- **职责**:
  - 架构设计
  - 任务拆解
  - 验收标准制定

---

## 💻 小研（研发工程师）

- **工具**: Claude Code + GLM-5
- **模型**: Claude 3.5 Sonnet / GLM-5
- **命令**: `claude --permission-mode bypassPermissions --print "[任务]"`
- **职责**:
  - 代码实现
  - 测试编写
  - 本地验证

---

## 🔍 小审（代码审核）

- **工具**: Codex CLI
- **模型**: GPT-4
- **命令**: `codex review --base main` 或 `codex --print "[审核任务]"`
- **职责**:
  - 代码审核
  - 质量把关
  - 安全检查

---

## 🎨 小美（设计）

- **工具**: Gemini 网页版（Pro 会员）/ Gemini CLI
- **模型**: Gemini Pro
- **能力**:
  - 文字创意
  - 图像生成（Google 出图）
  - 设计 Brief
- **职责**:
  - UI/UX 设计
  - 文案设计
  - 视觉方案

---

## ❌ 小运（已移除）

小运的工作已合并到小科：
- 文档整理
- 日报周报
- 归档管理

---

## 调度规则

| 任务类型 | 调度目标 | 执行方式 |
|----------|----------|----------|
| 项目创建/切换 | 小科 | 直接执行 |
| 状态查询 | 小科 | 直接执行 |
| 架构设计 | 小架 | 提示用户使用 ChatGPT |
| 代码研发 | 小研 | spawn Claude Code |
| 代码审核 | 小审 | spawn Codex |
| 设计/创意 | 小美 | 提示用户使用 Gemini 或 spawn Gemini CLI |
| 文档/日报 | 小科 | 直接执行 |

---

## Spawn 示例

### 小研（Claude Code）
```bash
claude --permission-mode bypassPermissions --print "
# 角色: 小研 - 研发工程师

# 任务: [任务描述]

# 项目背景:
[从 project.md 提取]

# 输出要求:
[具体输出格式]
"
```

### 小审（Codex）
```bash
codex --print "
# 角色: 小审 - 代码审核工程师

# 审核范围: [文件/模块]

# 审核标准:
[从架构文档提取]

# 输出要求:
审核报告格式
"
```

### 小美（Gemini CLI）
```bash
gemini "
# 角色: 小美 - 设计助手

# 设计需求: [需求描述]

# 输出要求:
- 设计 Brief
- 图像生成 Prompt
"
```
